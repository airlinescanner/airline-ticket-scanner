# Локализация (i18n)

## Обзор

Модуль локализации приложения Airline Ticket Scanner, реализованный с использованием `i18next` и `react-i18next`.

**Требования:** 9.1, 9.3, 9.6

## Поддерживаемые языки

- **Украинский (uk)** — основной язык (fallback)
- **Русский (ru)** — второй язык

## Структура файлов

```
src/i18n/
├── index.ts       # Инициализация i18next
├── uk.json        # Украинские переводы
├── ru.json        # Русские переводы
├── i18n.test.ts   # Тесты локализации
└── README.md      # Документация
```

## Использование

### В компонентах

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <Text>{t('demo.title')}</Text>
  );
}
```

### С интерполяцией

```typescript
const { t } = useTranslation();

// Простая интерполяция
t('demo.switchTheme', { theme: 'темну' })
// Результат: "Переключити на темну тему"

// Множественное число
t('airline.deleteWarning', { count: 5 })
// Результат: "Це вплине на 5 квитків"
```

### Переключение языка

```typescript
import i18n from './src/i18n';
import { saveLanguage } from './src/i18n';

// Переключить язык
await i18n.changeLanguage('ru');

// Сохранить выбор в AsyncStorage
await saveLanguage('ru');
```

## Структура переводов

Переводы организованы по секциям:

- **demo** — демонстрационный экран дизайн-системы
- **common** — общие элементы (кнопки, сообщения)
- **theme** — названия тем оформления
- **language** — названия языков
- **scanner** — экран сканирования
- **ticket** — работа с билетами
- **airline** — работа с авиакомпаниями
- **registration** — дата регистрации
- **notification** — уведомления
- **settings** — настройки приложения

## Добавление новых переводов

1. Добавьте ключ в `uk.json`:
```json
{
  "mySection": {
    "myKey": "Мій переклад"
  }
}
```

2. Добавьте соответствующий перевод в `ru.json`:
```json
{
  "mySection": {
    "myKey": "Мой перевод"
  }
}
```

3. Используйте в компоненте:
```typescript
t('mySection.myKey')
```

## Тестирование

Запустите тесты локализации:

```bash
npm test -- src/i18n/i18n.test.ts
```

Тесты проверяют:
- Корректность инициализации
- Наличие всех переводов
- Соответствие ключей в обоих языках
- Работу интерполяции

## Сохранение и восстановление

Выбранный язык автоматически сохраняется в `AsyncStorage` с ключом `@airline_scanner:language` и восстанавливается при следующем запуске приложения.

Если сохранённых настроек нет, применяется украинский язык по умолчанию (Требование 9.5).
